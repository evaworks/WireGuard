# DNS服务器配置指南

## 当前配置

默认配置使用：`8.8.8.8, 1.1.1.1`

- `8.8.8.8` - Google公共DNS
- `1.1.1.1` - Cloudflare公共DNS

## 配置分析

### 优点
- 无过滤，访问外网效果好
- 响应速度快
- 适合VPN使用场景
- 隐私保护较好

### 缺点（对国内用户）
- 访问国内网站时可能解析到国外CDN节点
- 国内网站访问速度可能变慢
- 部分国内服务可能无法正常访问

## 推荐配置方案

### 方案1：混合DNS（推荐，平衡国内外访问）

```
223.5.5.5, 8.8.8.8, 1.1.1.1
```

- `223.5.5.5` - 阿里云DNS（国内，速度快）
- `8.8.8.8` - Google DNS（国外）
- `1.1.1.1` - Cloudflare DNS（国外）

**优点**：
- 国内网站使用阿里云DNS，速度快
- 国外网站使用Google/Cloudflare DNS
- 系统会自动选择最快的DNS

### 方案2：纯国内DNS（适合主要访问国内网站）

```
223.5.5.5, 114.114.114.114
```

- `223.5.5.5` - 阿里云DNS
- `114.114.114.114` - 114DNS

**优点**：
- 国内网站访问速度快
- 解析准确

**缺点**：
- 访问外网时可能不如国外DNS

### 方案3：纯国外DNS（适合主要访问外网）

```
8.8.8.8, 1.1.1.1
```

**当前配置**，适合：
- 主要访问外网
- 需要绕过DNS过滤
- 对国内网站速度要求不高

## 如何修改DNS配置

### 方法1：在Mac WireGuard应用中修改

1. 点击"编辑"按钮
2. 找到"DNS 服务器"
3. 修改为您想要的DNS，例如：
   ```
   223.5.5.5, 8.8.8.8, 1.1.1.1
   ```
4. 点击"保存"
5. 断开并重新连接VPN

### 方法2：在服务器上修改（影响所有新客户端）

编辑 `manage.sh` 文件：

```bash
nano /root/vpn/server/manage.sh
```

找到第96行：
```bash
DNS = 8.8.8.8, 1.1.1.1
```

修改为：
```bash
DNS = 223.5.5.5, 8.8.8.8, 1.1.1.1
```

保存后，新添加的客户端会使用新DNS配置。

## 测试DNS效果

修改DNS后，测试效果：

```bash
# 测试DNS解析速度
time nslookup baidu.com 223.5.5.5
time nslookup google.com 8.8.8.8

# 测试当前使用的DNS
nslookup baidu.com
nslookup google.com

# 查看系统DNS配置
scutil --dns | grep 'nameserver\[0\]'
```

## 推荐配置

### 如果您主要访问外网
保持当前配置：`8.8.8.8, 1.1.1.1`

### 如果您需要平衡国内外访问
推荐：`223.5.5.5, 8.8.8.8, 1.1.1.1`

### 如果您主要访问国内网站
推荐：`223.5.5.5, 114.114.114.114`

## 注意事项

1. **DNS顺序很重要**：系统会按顺序尝试，第一个可用就使用
2. **VPN连接后生效**：修改DNS后需要重新连接VPN
3. **测试效果**：修改后测试国内外网站访问速度
4. **个人需求决定**：根据您的使用场景选择最适合的配置
