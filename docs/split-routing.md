# 分流配置：国内直连，国外走VPN

本文档说明如何配置WireGuard，实现国内网络直连，国外网络通过VPN访问。

## 配置原理

通过修改客户端的 `AllowedIPs` 配置，排除国内IP段，让国内流量不走VPN，国外流量走VPN。

## 方法一：使用排除列表（推荐，最简单）

### 在Mac WireGuard应用中配置：

1. **点击"编辑"按钮**
2. **找到"路由的 IP 地址（段）"**
3. **修改为以下配置**：

```
0.0.0.0/1, 128.0.0.0/1, ::/1, 8000::/1
```

这个配置的含义：
- `0.0.0.0/1` 和 `128.0.0.0/1` 组合起来等于 `0.0.0.0/0`（所有IPv4流量）
- 但这样配置可以让内网（10.x.x.x, 192.168.x.x等）不走VPN
- 外网流量走VPN

**注意**：这个方法会让大部分外网流量走VPN，包括部分国内网站。如果需要更精确的分流，使用方法二。

### 步骤2：在WireGuard中配置

由于WireGuard的 `AllowedIPs` 不支持排除语法，我们需要使用**反向思路**：

**配置所有流量走VPN，但通过路由表让国内IP直连**

这需要在Mac系统层面配置路由，比较复杂。

### 更实用的方法：使用简化配置

由于完整排除列表太长，建议使用以下简化配置：

```
0.0.0.0/5, 8.0.0.0/7, 11.0.0.0/8, 12.0.0.0/6, 16.0.0.0/4, 32.0.0.0/3, 64.0.0.0/2, 128.0.0.0/1, ::/1, 8000::/1
```

这个配置排除了部分国内IP段，但不是100%准确。

## 推荐方案：简化配置（平衡易用性和准确性）

### 在WireGuard应用中配置：

1. **点击"编辑"**
2. **修改"路由的 IP 地址（段）"为**：

```
0.0.0.0/5, 8.0.0.0/7, 11.0.0.0/8, 12.0.0.0/6, 16.0.0.0/4, 32.0.0.0/3, 64.0.0.0/2, 128.0.0.0/1, ::/1, 8000::/1
```

这个配置：
- ✅ 让大部分外网流量走VPN
- ✅ 部分国内IP段直连
- ⚠️ 不是100%准确，但足够使用

### 或者使用更简单的配置：

```
0.0.0.0/1, 128.0.0.0/1, ::/1, 8000::/1
```

这个配置：
- ✅ 外网流量走VPN
- ✅ 内网（10.x, 192.168.x等）直连
- ⚠️ 国内公网IP也会走VPN（但速度影响不大）

## 测试分流效果

配置后测试：

```bash
# 测试国内网站（应该直连，速度快）
ping baidu.com
curl -I https://www.baidu.com

# 测试国外网站（应该走VPN）
ping google.com
curl -I https://www.google.com

# 查看路由
netstat -rn | grep wg0
```

## 注意事项

1. **WireGuard的限制**：WireGuard的 `AllowedIPs` 不支持排除语法，只能通过路由表实现精确分流
2. **性能影响**：即使国内IP走VPN，由于服务器在香港，延迟增加通常不明显
3. **DNS解析**：确保DNS配置正确，使用 `8.8.8.8, 1.1.1.1` 或国内DNS如 `223.5.5.5`

## 总结

**最简单的方法**：
- 在WireGuard中设置：`0.0.0.0/1, 128.0.0.0/1, ::/1, 8000::/1`
- 这样内网直连，外网（包括国内公网）走VPN
- 对于大多数用户来说已经足够

**如果需要更精确的分流**：
- 考虑使用 Clash 等专业代理工具
- 或使用系统路由表手动配置（较复杂）
